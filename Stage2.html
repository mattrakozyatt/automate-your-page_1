Assignments in Python
Syntax: <name> = <expression> Example: speed_of_light = 2997924588
print
Prints expressions or variables. When using print with a string, you can combine, or concatenate, segments of text with the "+" sign. Example: print 'Hello' + ' world!' <-- will print: Hello world! You cannot combine numbers with strings. Example: print 'hello' + 13 <-- This will result in an error It is possible to multiply strings. Example: print 'hello ' * 3 <-- will print: hello hello hello
String indexing
String indexing evaluates the characters of a string. Example: 'udacity'[3] - produces the result 3 (fourth character in the string, counting from 0) Index numbering starts with 0. Indexing also works on variables. Example: name = "Matthew" print name[4] <-- will print "h" When using negative numbers in the indexing evaluation, the character count starts from the end of the string starting with -1. Example: "pomade"[-1] <-- result with "e"
Selecting subsequences of strings
Syntax: <string>[<start expression>:<stop expression>] The first expression in the evaluation is the starting position, and the second expression is the stop. The result will be a string that is a subsequence of the characters in the string, starting from the start position, and ending at the stop position -1. Example: name = 'awesome' print name[2:5] <-- will print 'eso' Selecting a subsequence without a 'stop' value will result with an evaluation with all characters starting at the 'start' character, and will include all remaining characters through the end of the string. Example: statement = "OK, let's begin" print statement[10:] <-- will print "begin" When selecting a subsequence without a 'start' value will result with an evaluation with all characters from the beginning of the string, and ending at the 'stop' position -1. Example: name = "Harkness" print name[:5] <-- will print "Harkn"
Finding strings within strings using .find
Syntax: <string>.find(<string to find>) Results in a number that gives the position in the searched string, of the first character of the string that is being sought. ** If the sought string is not found, the result is -1. Example: name = 'hello' print name.find('ll') <-- will print the result '2'
Finding strings with a starting position
Syntax: <string>.find(<string>,<expression>) The expression will give the start position in the searched string, where the target string appears at, or after, the start position. Example: name = "Okay, Okay, Okay" print name.find('O',3) <-- will print the result '6'
Functions
Also called procedures. A series of statements which returns some value to a caller. It can also be passed zero or more arguments which may be used in the execution of the body. Syntax: def <function>(<input>,<input>,<input>....): <operation> Inputs are also called 'operands' and 'arguments'. Example: def rest_of_string(string): return string[1:] print rest_of_string('audacity') <-- will print 'udacity' When a function is called, the execution jumps to the function. It assigns the parameters, or the values passed as input. In the body of the procedure, the input(s) are acted upon by the operation. Using 'return' in the operation, tells the function to return the results to where the function was called.
Control Flows and Loops: if and while
if - controls which code gets executed when Syntax: if <test expression>: <block> Example: def bigger(a,b): If a > b: return a return b 'else' after an 'if' will execute the <block> under 'else' when the 'if' statement is False. Example: def bigger(a,b): if a < b: return a else: return b while - loops code to perform the same task many times Syntax: while <test expression>: <block> As long as the test expression in a 'while' is True, the function will repeat. Example: i = 0 while i < 10: print i i = i + 1 'break' exits the loop if conditions are True. Example: i = 1 while True: if I > 20: break print i i = i + 1
Using 'or' in 'if' statements
Syntax: <expression> or <expression> or <expression>... If the first expression evaluates to True, the value is True, and the second expression is not evaluated. If the value is False, then the next expression is tested.
Making decisions
Results of comparisons are not numbers. Results of comparisons are Boolean values: True or False. Operators for comparison '<' less than '>' greater than '==' equal '!=' does not equal
Debugging
If the code does not work, isolate it by commenting it out. If you were construction your code by copying code from a known good program, replace the broken code with the good example. This will help to identify the error. Bugs happen to all programmers. It is not important to avoid bugs, but it is important to know what to do when you encounter an error. Copy code into a separate file to preserve versions.
Debugging Strategy recap
1) Examine error messages when programs crash 2) Work from example code 3) Make sure examples work 4) Check(print) intermediate results 5) Keep and compare versions of code

Lists
A list is a sequence of anything. A list can include characters, strings, numbers, and other lists. Both lists and strings can be assigned to variables. A string is identified by the encapsulation of characters by quotation marks(either single or double). A list is identified by its encapsulation by brackets []. The elements in a list are separated by commas. Strings vs. Lists Example: strings lists s = 'yabba' p = ['y','a','b','b','a'] s[0] - 'y' p[0] - 'y' s[2:4] - 'bb' p[2:4] - ['b','b'] A string is a sequence of characters(the only thing you can put in a string is a character) A list can be empty Example: [] A list can have one element Example: [3] A list can contain lists(called nested lists) Example: ['apple',3,'oranges',27[1,2['alpha','beta']]] List mutation changes the value of the list after it is created. Example: p = ['y','e','l','l','o'] p[0] = 'H' <-- changes the 'y' to an 'H' in position 0 A list can change assignment Example: q = p <-- assigns the list p to q q[4] = '!' <-- This changes the value in both p and q
List operations
Append - Adds a new element to a list Syntax: <list>.append(<element>) Example: alpha = ['A','B','C'] alpha.append('D') <-- resulting list is ['A','B','C','D'] Concatenating lists with '+' Example: [0,1] + [2,3] <-- resulting list is [0,1,2,3] len - Length of object Syntax: len(<list or string>) Example: len([0,1]) <-- resulting value is 2 len(['a',['b',['c']]] <-- resulting value is 2 len('udacity') <-- resulting value is 7
For loops
Syntax: for <name> in <list>: <block> For each element in the list, we are going to assign the element to the <name>, and evaluate the <block>. And were are going to do this for each element in the list(in order). Example: def print_all_elements(p): For e in p: print e
Indexing lists
The output of index is the first position of the found value in the list. If the <value> is found in the list, returns the first position where <value> is found in <list>. Otherwise produces and error. Syntax: <list>.index(<value>) Example: nums=[6,7,8,9] print nums.index(8) <-- will print the value '2'
in
If <value> is in the list, result is True, otherwise the result is False. Syntax: <value> in <list>
not in
If <value> is not in the <list>, result is True, otherwise the result is False. Syntax: <value> not in <list>
How to Solve Problems
The first thing you should do to solve a problem is make sure you understand the problem. Example of a problem: Given your birthday, and the current date, calculate your age in days. The input(set) of this it the first part of the problem: "birthday, and the current date". The output is the second part of the problem statement: "age in days". The solution to a problem is a procedure that can take any inputs in the (set), and produces output, the desired output that satisfies the relationship we want. A problem is defined by the set of possible inputs(this is usually an infinite set for most interesting problems), and the relationship between those inputs and their desired outputs. Solution Input -> Procedure -> output
